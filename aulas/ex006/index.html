Operadores

Aritméticos

5 + 2 = 7 -> (soma)

5 - 2 = 3 -> (subtração)

5 * 2 = 10 -> (multiplicação)

5 / 2 = 2.5 -> (divisão)

5 % 2 = 1 -> (mod(resto))

5 ** 2 = 25 -> (potência)

Ordem de Precedência dos Aritméticos

1º -> ()

2º -> **

3º -> * / %

4º -> + -

Atribuição

Atribuições simples

var a = 5 + 3 -> 8

var b = a % 5 -> 3

var c = 5 * b ** 2 -> 45

var d = 10 - a / 2 -> 6

var e = 6 * 2 / d -> 2

var f = b % e + 4 / e -> 3

Auto-atribuições

var n = 3 -> n = n + 4

n = n + 4 (simplificado -> n += 4 (n recebe ele mesmo + 4))

Incremento

x += 1 ou x ++

x ++ recebe x + 1

x -= 1 ou x --

x -- recebe x - 1

Relacionais

Expressões com operador relacional SEMPRE vai gerar um resultado booleano (true ou false)

ex: 5 > 2 == true

< menor que

> maior que

>= maior ou igual

<= menor ou igual

== igual

!= diferende

Identidade

5 == 5 true

5 == '5' true (o js não compara o tipo do valor, so seu peso portanto 5 e '5' são iguais)

5 === '5' false (=== verifica se são identicos (mesmo valor e mesmo tipo))

5 === 5 true

5 !== '5' true (!== verifica se a identidade é diferente)

Ordem de Precedência dos Relacionais

Não tem, resolve da esquerda para a direita

Lógicos

Expressões lógicas SEMPRE vai gerar um resultado booleano (true ou false)

! negação ( not )

not True = False

not False = True

&& conjunção ( and )

So vai ser TRUE se todas forem TRUE

True and True = True

False and True = False

True and False = False

False and False = False

|| disjunção ( or )

Se pelo menos uma for TRUE vai ser TRUE

True or True = True

True or False = True

False or True = True

False or False = False

Ordem de Precedência dos Lógicos

1º -> not

2º -> and

3º -> or

Ternário

? :

teste ? true : false

ex: media >= 7.0 ? "Aprovado" : "Reprovado"

Vai aparecer "Aprovado" se for TRUE e "Reprovado" se for FALSE

Ordem de Precedência entre os operadores

1º -> Aritméticos

2º -> Relacionais

3º -> Lógicos

4º -> Ternário
